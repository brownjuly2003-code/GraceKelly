## batch-108: GK-UI-flakiness-1 — Sonar auto-route retry

### Goal
Fix MODEL_MISMATCH circuit-breaker cascade caused by Perplexity auto-routing requests to Sonar
when Claude Sonnet 4.6 was explicitly selected.

### Context
After the threadpool fix (batch-107 / commit a7b6fe3), RAG 20-case regression exposed two new
GK-side flakiness modes. The submit-button timeout (`Locator.click: Timeout 5000ms exceeded`) was
already fixed inline in `playwright_driver.py:665` by adding `force=True` (that commit is staged,
not yet committed — do NOT re-apply). This spec covers the remaining blocker.

**What happens:** Perplexity UI sometimes auto-routes to Sonar after the browser adapter clicks
Claude Sonnet 4.6 in the model picker. `_model_matches_expected` returns False → MODEL_MISMATCH
failure. After 3 consecutive mismatches the circuit breaker opens, blocking all subsequent
requests in the pipeline run.

**Root cause observation:** The Perplexity classifier overrides the selection AFTER the click
succeeds — so re-selecting is the right fix, not changing the selector. Re-selecting up to 2
extra times with a short delay gives the UI time to settle on the intended model.

**File to modify:** `src/gracekelly/adapters/browser/perplexity.py`
Relevant section: `_execute_inner`, lines ~140-158 (the block that calls `select_model` then
checks `_model_matches_expected`).

**Related constants to respect:**
- `PerplexityBrowserAdapter._DEDICATED_THREAD_PREFIX` — we are already on the dedicated thread
  inside `_execute_inner`, so `time.sleep()` is safe.
- `FailureCode.MODEL_MISMATCH` — still the correct failure code when all retries are exhausted.

### Deliverables

1. **`src/gracekelly/adapters/browser/perplexity.py`** — wrap the `select_model` call + mismatch
   check in a small retry loop:
   - `_MODEL_SELECT_RETRIES = 2` (module-level constant or class constant, NOT an env var)
   - On mismatch (before last attempt): `logger.warning(...)` with attempt number, actual label,
     expected label; `time.sleep(1.5)`; loop back to `select_model`.
   - After the last attempt still mismatched: return `self._failure(...)` exactly as before.
   - On match: `break` from the loop and continue with the rest of `_execute_inner`.
   - No new public API changes. No changes to `BrowserAutomationPort` or `automation.py`.

2. **`tests/test_browser_adapter.py`** — add two test cases to the existing suite:
   - `test_sonar_override_retried_then_succeeds`: mock `select_model` to return
     `BrowserModelSelection(requested_label="Claude Sonnet 4.6", actual_label="Sonar")` for the
     first 2 calls, then return a matching label; assert execution succeeds (status COMPLETED) and
     that `select_model` was called 3× total.
   - `test_sonar_override_exhausts_retries`: mock `select_model` to always return Sonar; assert
     execution returns MODEL_MISMATCH failure and `select_model` was called exactly 3× total.
