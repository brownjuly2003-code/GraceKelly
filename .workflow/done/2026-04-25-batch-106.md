# Batch 106 — GraceKelly :8011 always-on (Windows user session autostart)
Date: 2026-04-25
Tier: S

## Context

Сейчас V2 GraceKelly запускается ad-hoc через `uvicorn ...`. Все 3 клиента (RAG, agent_toolkit, juhub) от него зависят: RAG — async ходящие в V2, juhub — daily debate cron 08:30, agent_toolkit — интеграционные тесты + ручные runs. Если Юлия не запустила uvicorn до 08:30 — daily debate skip; если не запустила в момент RAG-запроса — fallback на ollama.

Цель — **GraceKelly :8011 крутится в user session начиная с logon**, без явного запуска руками. Single-user local scope, поэтому Windows Task Scheduler "At log on" — самый простой подход (NSSM overkill, headless service не нужен — V2 не network-exposed beyond localhost).

## Task: SETUP-windows-autostart-artefact

scope: создать в `scripts/win-autostart/` следующее:

1. **`gracekelly_uvicorn.bat`** — запускалка:
   - `cd D:\GraceKelly`
   - `set GRACEKELLY_EXECUTION_PROFILE=dry-run` (default; см. note ниже)
   - `.venv\Scripts\python.exe -m uvicorn gracekelly.main:create_app --factory --host 127.0.0.1 --port 8011 >> %LOCALAPPDATA%\GraceKelly\uvicorn.log 2>&1`
   - Создать `%LOCALAPPDATA%\GraceKelly\` если не существует (`if not exist ... mkdir ...`).
   - Note по profile: default `dry-run` безопасен для тестов; для реальной работы Юлия меняет env на `hybrid` через отдельный механизм (см. Task 2 ниже).

2. **`gracekelly_autostart.xml`** — Windows Task Scheduler XML описание:
   - Trigger: `LogonTrigger`, текущий user (Juлin domain user).
   - Action: `gracekelly_uvicorn.bat` (с absolute path или с `%~dp0` resolve).
   - Settings: `RestartOnFailure` (3 retries, 5min interval), `DisallowStartIfOnBatteries: false`, `StopIfGoingOnBatteries: false`, `StartWhenAvailable: true`, `Hidden: false` (чтобы Юлия видела процесс если что), `MultipleInstancesPolicy: IgnoreNew`.
   - **Не указывать пароль user'а** — `LogonType=InteractiveToken` чтобы пароль не требовался при импорте.

3. **`install_autostart.bat`** — one-command установщик:
   - `schtasks /Create /TN "GraceKelly Autostart" /XML "%~dp0gracekelly_autostart.xml" /F`
   - Echo success / failure summary.
   - Требует Admin elevation — `install_autostart.bat` либо явно вызывает `runas`, либо в начале проверяет `net session >nul 2>&1` и просит запускать от Admin (не проксирует runas сам — это user-decision).

4. **`uninstall_autostart.bat`** — отключить:
   - `schtasks /Delete /TN "GraceKelly Autostart" /F`

5. **`scripts/win-autostart/README.md`** — операционная справка:
   - Что это, зачем.
   - Как установить (run `install_autostart.bat` от Admin).
   - Как поменять execution profile (отредактировать `.bat` либо positional `set` override).
   - Где лог (`%LOCALAPPDATA%\GraceKelly\uvicorn.log`).
   - Как проверить статус (`schtasks /Query /TN "GraceKelly Autostart" /V /FO LIST`).
   - Как уничтожить (`uninstall_autostart.bat` от Admin).
   - Troubleshooting: `:8011 already in use`, `python not in PATH` etc.

done_when:
- 5 файлов созданы.
- README прошёл проверку: можно отдать другому Windows-пользователю и развернуть с нуля без вопросов.
- `.bat`/XML — без syntax errors (XML парсится, .bat не имеет typos в путях).
- `git add scripts/win-autostart/` + commit:
  - `batch-106: SETUP-windows-autostart-artefact — Task Scheduler XML + .bat installers + README`.

**Не выполнять `install_autostart.bat` в рамках batch'а.** Это user-action, требует Admin. CX только создаёт artefacts, Юлия запускает когда готова.

## Task: SUPPORT-execution-profile-toggle

scope: добавить в `scripts/win-autostart/` дополнительный helper `set_profile.bat`:
- Принимает 1 arg: `dry-run` / `api-only` / `hybrid`.
- Записывает значение в `%LOCALAPPDATA%\GraceKelly\profile.env` (одна строка `GRACEKELLY_EXECUTION_PROFILE=<value>`).

Цель — Юлия может переключать profile **без edit'а .bat**: `set_profile.bat hybrid` → restart task → новый profile активен.
