## Task: 109-1-usage-telemetry
goal: Append per-request line to JSONL log so a 30-day usage audit can answer which endpoints/models are actually used.
scope: src/gracekelly/middleware.py, src/gracekelly/main.py, src/gracekelly/config.py, .env.example, tests/test_middleware_usage_telemetry.py
done_when:
- New `setup_usage_telemetry(app, log_path)` middleware appends one JSON object per request after `call_next`, mirroring the pattern of `setup_request_metrics` (file: src/gracekelly/middleware.py, lines 83-94).
- Each line carries: `ts` (UTC ISO8601), `endpoint` (normalised via `_normalize_endpoint`), `method`, `status`, `duration_ms` (int), `request_id` (header `X-Request-ID` if present, else uuid4), `prompt_hash` (sha256 of body for POST endpoints whose path starts with `/api/v1/orchestrate|consensus|compare|debate|smart|smart/v2|batch|pipeline`, else null) — body must be re-readable downstream (cache the body, do not consume the stream).
- New env vars in `Settings`: `usage_telemetry_enabled: bool = False`, `usage_telemetry_path: str | None = None` (default-resolved to `<repo_root>/logs/usage.jsonl` when enabled and unset). Loaded via existing `_env_bool` / `os.getenv` helpers in `config.py`. Documented in `.env.example`.
- Wired in `create_app` AFTER `setup_correlation_id` (so the request_id header is already set) but BEFORE `setup_rate_limiting`. No-op when disabled.
- Failure to write the JSONL line never raises — log a single WARNING `usage_telemetry.write_failed` per process and continue.
- New tests cover: enabled-path writes one line per request with all 7 keys, disabled-path writes nothing, prompt_hash present only for the listed POST routes, write failure does not break the response.
- `ruff check src tests` clean; `mypy src --strict` 0 new errors; `pytest tests/test_middleware_usage_telemetry.py -v` green; existing tests do not regress.

## Task: 109-2-selectors-recon-cron
goal: Weekly Windows scheduled task that captures Perplexity selectors, diffs against a baseline snapshot, and writes a notification when drift is detected — so UI breaks are caught before they hit a live request.
scope: scripts/recon_weekly.py, scripts/win-autostart/install_recon_cron.bat, scripts/win-autostart/recon-task.xml, scripts/win-autostart/README.md, tests/test_recon_weekly.py
done_when:
- New `scripts/recon_weekly.py` is a runnable CLI: invokes the existing `tools/capture_perplexity_recon.py` capture path (import it, do not duplicate selector logic), writes the fresh snapshot to `.workflow/state/perplexity-selectors-latest.json`, and `diff`s it against `.workflow/state/perplexity-selectors-baseline.json`.
- First run (no baseline present): copies latest to baseline, exits 0 with `recon.baseline_created` log line — never reports false drift.
- Subsequent runs: structural diff (added/removed/changed selector keys); on any diff appends one line to `logs/recon-drift.jsonl` (`{ts,added:[],removed:[],changed:[]}`) AND writes `.workflow/state/perplexity-selectors-drift.flag` with the same payload; exits 1. No diff: exits 0, no log line, removes the flag if it exists.
- New `scripts/win-autostart/recon-task.xml` is a Windows Task Scheduler XML for weekly Friday 03:00 local trigger, action = `python.exe scripts\recon_weekly.py` from `D:\GraceKelly\`. Use the existing autostart XML in the same folder as a structural template — match user account placeholder convention from batch-108 (`__USER__`).
- New `install_recon_cron.bat` registers the XML via `schtasks /create /xml ... /tn "GraceKelly Selectors Recon"` (admin); a paired `uninstall_recon_cron.bat` removes it. Both follow the style of the existing `install_autostart.bat` / `uninstall_autostart.bat`.
- `scripts/win-autostart/README.md` gains a short subsection at the bottom: when it runs, where the drift flag/log live, how to acknowledge drift (delete flag, regenerate baseline).
- Tests cover: first-run baseline creation, no-diff exit 0, drift produces JSONL line + flag + exit 1, flag removed when drift resolves. Use a temp dir + monkeypatched capture function (do NOT spawn a real browser in tests).
- `ruff check src tests scripts` clean; `mypy scripts/recon_weekly.py --strict` 0 errors; `pytest tests/test_recon_weekly.py -v` green.
