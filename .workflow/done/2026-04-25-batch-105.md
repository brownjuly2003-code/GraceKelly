# Batch 105 — end-to-end ecosystem smoke script
Date: 2026-04-25
Tier: S

## Context

3 клиента (`RAG_Support_Assistant`, `agent_toolkit`, `juhub`) теперь висят на V2 GraceKelly (:8011). Любое изменение V2 потенциально ломает любого из них. Сейчас diagnostic — это вручную: boot uvicorn, ходить по каждому проекту, запускать там свой smoke. Долго и забывается.

Цель — **один command в `D:\GraceKelly`**, который последовательно гоняет smoke всей экосистемы и выдаёт PASS/FAIL summary. Этот же скрипт можно встроить позже в pre-release gate.

## Task: SCRIPT-ecosystem-smoke

scope: новый файл `scripts/ecosystem_smoke.py` (либо `scripts/ecosystem_smoke.ps1` если bash слишком неудобен под Windows — на усмотрение CX). Самодостаточный, без новых deps вне `httpx` (уже используется в проекте).

Поведение:

1. **Pre-flight**: проверить `:8011/healthz/ready` — 200. Если нет — `print("ERROR: GraceKelly :8011 not reachable, boot uvicorn first")`, exit 1. **Не пытаться запустить uvicorn сам** — script проверяет, не оркеструет.

2. **Step 1 — direct V2 smoke** (sanity что сам V2 жив):
   - `POST /api/v1/smart {"prompt":"2+2"}` → 200, `model_id` ≠ `mistral-small`, `answer` без `provider_unavailable`/`Mistral`.
   - `POST /api/v1/orchestrate {"prompt":"2+2","model":"claude-sonnet-4-6","reliability_level":"quick"}` → 200, `status: "completed"`, `execution_mode: "dry-run"` (если profile=dry-run).
   - PASS/FAIL line per step.

3. **Step 2 — RAG_Support_Assistant smoke**:
   - Если `D:/RAG_Support_Assistant/scripts/gracekelly_smoke.py` существует и `:8000` слушает — `subprocess.run(["python", "D:/RAG_Support_Assistant/scripts/gracekelly_smoke.py"])`, capture exit code + last 30 строк stdout.
   - Если RAG не запущен — `SKIP: RAG :8000 not reachable`.
   - Если smoke-файл отсутствует — `SKIP: RAG smoke harness not found`.

4. **Step 3 — agent_toolkit integration**:
   - `subprocess.run(["uv", "run", "pytest", "tests/integration/", "-q", "--no-header"], cwd="D:/agent_toolkit")`.
   - Capture pytest summary line (`N passed, M skipped/failed`).
   - PASS если 0 failed; SKIP если `uv` отсутствует; FAIL иначе.

5. **Step 4 — juhub debate dry-run**:
   - С env `GK_DRY_RUN=1` (юзается в juhub config per migration) — `subprocess.run(["python", "-m", "juhub.backend.scheduler", "--now", "--dry-run"], cwd="D:/Perplexity_Orchestrator2", env={...os.environ, "GK_DRY_RUN": "1"})`.
   - Если scheduler `--dry-run` flag отсутствует — fallback: `python -m juhub.backend.scheduler --now`, ожидать что dry-run сам активируется через env.
   - Capture exit code.
   - PASS если 0; SKIP если juhub отсутствует; FAIL иначе.

6. **Final summary table**:
   ```
   Step | Component       | Status | Detail
   1.1  | V2 /smart       | PASS   | model=claude-sonnet-4-6
   1.2  | V2 /orchestrate | PASS   | status=completed
   2    | RAG smoke       | PASS   | 3 PASS / 5 SKIP / 0 FAIL
   3    | agent_toolkit   | PASS   | 76 passed
   4    | juhub debate    | SKIP   | scheduler not present
   ```
   Exit 0 если **все** steps PASS или SKIP; exit 1 если хоть один FAIL.

7. **CLI flags**:
   - `--gracekelly-url` (default `http://127.0.0.1:8011`)
   - `--rag-url` (default `http://127.0.0.1:8000`)
   - `--skip-rag` / `--skip-agent-toolkit` / `--skip-juhub` для опциональной выборки шагов
   - `--verbose` для полного stdout subprocess'ов
