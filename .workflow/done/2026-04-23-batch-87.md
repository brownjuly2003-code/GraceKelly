- Playwright `text=` engine matches full text (case-insensitive, normalized whitespace); `has-text` — partial substring. Оба допустимы в этом контексте. В Orchestrator2 используют `text={name}` для primary и `:has-text("{name}")` для fallback — повторить ту же семантику.
- Helper'ы приватные (`_find_option_*`), держать в одном файле с `select_model`. Не выносить в отдельный модуль.
- `has-text` принимает string literal с кавычками: `[role="menuitem"]:has-text("claude-sonnet-4-6")` — экранирование не нужно для валидных provider_model_id (все ASCII, без `"`).
- Если `model_menu_item_selector` содержит `button` без role-filter (итого `button:has-text(name)`) — это широкий селектор, может зацепить composer chip.
