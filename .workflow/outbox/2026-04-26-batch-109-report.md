## Tasks

### 109-1 — usage telemetry middleware (R1)

`setup_usage_telemetry(app, enabled, log_path)` — opt-in JSONL append per HTTP request to `<repo>/logs/usage.jsonl` (or `GRACEKELLY_USAGE_TELEMETRY_PATH`). Records carry `ts` (UTC ISO8601), `endpoint` (UUID-normalised), `method`, `status`, `duration_ms`, `request_id`, `prompt_hash` (sha256 of body for orchestration POST routes only). Body is replayed via `request._receive` so downstream handlers consume it intact. Write failures are logged once and degrade silently.

Wired in `create_app` as the outermost middleware so rate-limited and error responses are still recorded. Toggle via `GRACEKELLY_USAGE_TELEMETRY_ENABLED=true`. Default disabled.

### 109-2 — selectors weekly recon cron (R4)

`gracekelly-recon-weekly` (entry point) — captures live Perplexity DOM via existing `capture_perplexity_recon`, normalises buttons + model menu + manifest flags into a snapshot dict, diffs against `.workflow/state/perplexity-selectors-baseline.json`. First run seeds the baseline; subsequent runs exit 0 on no-diff (and clear any stale flag), exit 1 on drift after appending to `logs/recon-drift.jsonl` and writing `perplexity-selectors-drift.flag`.

Windows Task Scheduler: `recon-task.xml` is a CalendarTrigger weekly Friday 03:00 task with 15-minute execution timeout. `install_recon_cron.bat` substitutes `%USERDOMAIN%\%USERNAME%` into the placeholder and registers via `schtasks /Create /XML`. `uninstall_recon_cron.bat` removes it.
